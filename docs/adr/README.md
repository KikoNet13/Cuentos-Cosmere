# ADR Directory

Architecture Decision Records (ADRs) capture durable technical choices.

## When to Create an ADR
Create ADRs only for architectural decisions, including:
- repository governance and process contracts
- data persistence policy
- import/export contract changes
- long-lived model/schema direction

## Statuses
- proposed
- accepted
- superseded
- obsolete

## Naming
`NNNN-short-kebab-title.md` (example: `0001-repo-governance.md`).

## Minimum Sections
- Context
- Decision
- Consequences
- Status
- Date

Use `docs/adr/TEMPLATE.md` as the starting point.
